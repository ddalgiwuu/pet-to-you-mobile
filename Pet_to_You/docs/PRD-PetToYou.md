## 1. 프로젝트 개요

### 1.1 프로젝트 비전 및 목표

#### 1.1.1 핵심 비전

**비전 선언문**:
> "모든 반려동물이 필요한 의료, 돌봄, 사랑을 받을 수 있는 투명하고 신뢰할 수 있는 생태계를 만든다"

Pet to You는 대한민국 반려동물 서비스 시장의 **정보 비대칭성과 신뢰도 문제**를 해결하는 통합 플랫폼입니다.

**3가지 핵심 서비스**:
1. **병원 예약**: 의료 서비스 접근성 개선, 건강 기록 통합 관리
2. **유치원 예약**: 투명한 가격과 인증된 시설로 안심 돌봄
3. **입양 플랫폼**: 신뢰할 수 있는 보호소와 투명한 입양 절차

**해결하는 사회적 문제**:
- 애견 유치원 중도해지율 40% (신뢰도 부족)
- 가짜 보호소 파양비 착취 (연 30억원+ 피해)
- 유기동물 입양율 40% (낮은 신뢰도로 인한 펫샵 구매 증가)
- 연 10만+ 마리 유기동물 중 60% 생명 위협

#### 1.1.2 비즈니스 목표

**단기 목표 (6개월) - MVP 출시**:
- MAU 50,000명 달성
- 제휴 파트너 확보:
  - 동물병원 50개소
  - 애견 유치원 30개소
  - 보호소 20개소
- 월 예약/신청 건수:
  - 병원 예약: 3,000건
  - 유치원 예약: 2,000건
  - 입양 신청: 500건
- 사용자 만족도 4.3/5.0 이상

**중기 목표 (1년) - 시장 확대**:
- MAU 250,000명 달성
- 제휴 파트너 확보:
  - 동물병원 200개소
  - 애견 유치원 150개소
  - 보호소 100개소
- 월 예약/신청 건수:
  - 병원: 15,000건
  - 유치원: 10,000건
  - 입양: 2,500건
- 월 매출 6-7억원 달성
- Series A 투자 유치 (100억원)

**장기 목표 (2-3년) - 시장 주도**:
- MAU 1,200,000명 달성
- 시장 점유율:
  - 병원 예약: 5%
  - 유치원 예약: 10%
  - 입양 플랫폼: 20%
- 연 매출 230억원 달성
- Phase 2 고도화:
  - 실시간 모니터링 도입
  - AI 매칭 시스템
  - 원격 진료 서비스
- Phase 3 확장:
  - 해외 진출 (일본, 동남아)
  - 펫 보험 연동
  - 커뮤니티 플랫폼

#### 1.1.3 성공 지표 (KPIs)

**사용자 지표**:
- **MAU (Monthly Active Users)**: 월간 활성 사용자 수
- **리텐션율**: D1 40%, D7 25%, D30 15%
- **예약 완료율**: 예약 시도 대비 완료 비율 70% 이상
- **NPS (Net Promoter Score)**: 40 이상

**비즈니스 지표**:
- **예약 건수**: 월 예약 건수 및 성장률
- **병원 파트너십**: 제휴 병원 수 및 지역 커버리지
- **매출**: 예약 수수료 및 프리미엄 기능 구독
- **CAC/LTV 비율**: 1:3 이상 (고객 획득 비용 대비 생애 가치)

**운영 지표**:
- **예약 취소율**: 10% 이하 유지
- **앱 크래시율**: 0.5% 이하
- **API 응답 시간**: 평균 200ms 이하
- **고객 지원 응답 시간**: 평균 2시간 이내

### 1.2 타겟 사용자

#### 1.2.1 주요 페르소나 (Primary Persona)

**페르소나 1: 직장인 김민지 (29세, 여성)**

**배경**:
- 직업: IT 회사 근무 (9-6시)
- 반려동물: 3살 웰시코기 "초코"
- 거주지: 서울 강남구
- 소득: 월 400만원

**Pain Points**:
- 근무 시간 중 병원 예약 전화가 어려움
- 야간 응급 상황 시 24시 병원 찾기 어려움
- 예방접종, 건강검진 스케줄 관리 불편
- 병원마다 다른 진료 기록 관리의 어려움

**니즈**:
- 언제든 모바일로 간편하게 예약
- 내 위치 기반 가까운 병원 검색
- 반려동물 건강 기록 통합 관리
- 예약 알림 및 리마인더

**사용 시나리오**:
> "퇴근 후 초코가 기침을 심하게 해서 걱정이 됩니다. 앱을 열어 '호흡기 질환' 전문 병원을 검색하고, 내일 아침 출근 전 시간대로 예약합니다. 예약 확인 알림을 받고 안심합니다."

**페르소나 2: 은퇴자 박영수 (65세, 남성)**

**배경**:
- 직업: 은퇴 (전 공무원)
- 반려동물: 10살 페르시안 고양이 "나비", 7살 비글 "복이"
- 거주지: 경기 수원시
- 소득: 연금 월 250만원

**Pain Points**:
- 다중 반려동물 관리의 복잡성
- 노령 반려동물의 잦은 병원 방문
- 과거 진료 기록 추적의 어려움
- 새로운 기술 적응의 어려움

**니즈**:
- 간단하고 직관적인 인터페이스
- 여러 반려동물의 예약 및 기록 통합 관리
- 과거 진료 이력 쉽게 조회
- 큰 글씨와 명확한 버튼

**사용 시나리오**:
> "나비의 정기 검진 날짜가 다가왔다는 알림을 받습니다. 앱에서 지난번 방문했던 병원을 선택하고, 나비의 과거 진료 기록을 확인한 후 예약을 진행합니다."

#### 1.2.2 부가 페르소나 (Secondary Persona)

**페르소나 3: 대학생 이수현 (22세, 여성)**

**배경**:
- 직업: 대학생 (수의학과 3학년)
- 반려동물: 1살 코숏 "모모"
- 거주지: 서울 관악구 (원룸)
- 소득: 아르바이트 월 100만원

**Pain Points**:
- 제한된 예산으로 저렴한 병원 찾기
- 야간 응급 상황 대처 정보 부족
- 반려동물 초보자로서 정보 부족

**니즈**:
- 가격 비교 및 저렴한 병원 필터
- 응급 상황 대응 가이드
- 커뮤니티 및 정보 공유

**페르소나 4: 맞벌이 부부 정수진 (35세, 여성)** ⭐ 유치원 이용자

**배경**:
- 직업: 마케팅 매니저 (9-7시)
- 배우자: IT 개발자 (재택 근무)
- 반려동물: 2살 포메라니안 "뽀미"
- 거주지: 서울 마포구 (아파트)
- 소득: 가구 월 900만원

**Pain Points**:
- 출근 시 반려견 홀로 두기 불안
- 유치원 가격 정보 불투명 (전화 상담 필수)
- 시설 안전성 확인 어려움
- 실시간으로 뽀미 상태 확인 불가
- SNS 정보만으로 선택해야 하는 불안감

**니즈**:
- 투명한 가격 정보와 비교
- 인증된 시설과 전문 교사
- 실시간 CCTV 또는 활동 리포트
- 간편한 예약 및 결제
- 픽업/드롭오프 서비스

**사용 시나리오**:
> "출장이 잡혀 뽀미를 3일간 맡길 곳이 필요합니다. 앱에서 집 근처 '인증됨' 배지가 있는 유치원을 검색하고, 가격표를 비교한 후 리뷰가 좋은 곳을 선택합니다. 예약 완료 후 매일 뽀미의 활동 사진을 받아 안심합니다."

**페르소나 5: 동물 보호 활동가 이지은 (28세, 여성)** ⭐ 입양 희망자

**배경**:
- 직업: NGO 활동가
- 거주지: 서울 마포구 (빌라, 마당 있음)
- 소득: 월 280만원
- 반려동물: 유기견 "구름이" (입양 2년차)

**Pain Points**:
- 신뢰할 수 있는 보호소 찾기 어려움
- 가짜 보호소 파양비 착취 사례 인지
- 입양 가능 동물의 정확한 건강 정보 부족
- 보호소 방문 시간/거리 제약
- 입양 후 문제 발생 시 도움 받을 곳 없음

**니즈**:
- 인증된 보호소만 보기
- 동물 건강 정보 투명 공개
- 온라인으로 입양 절차 진행
- 입양 후 상담 및 지원
- 가짜 보호소 신고 기능

**사용 시나리오**:
> "새로운 가족을 찾고 싶어 앱을 엽니다. '인증된 보호소만' 필터를 켜고, 성격이 온순한 중형견을 검색합니다. 보호소의 사업자등록증과 입양 성공률을 확인하고, 영상으로 동물을 미리 본 후 입양 신청을 합니다. 상담 예약까지 앱에서 완료합니다."

### 1.3 핵심 가치 제안

#### 1.3.1 사용자 Pain Points

**현재 문제점**:

1. **불편한 예약 절차**
   - 전화 예약만 가능한 병원 다수
   - 근무 시간 중 예약 전화 불가
   - 대기 시간 불확실성

2. **정보 분산**
   - 여러 병원에 흩어진 진료 기록
   - 예방접종 일정 관리 어려움
   - 과거 진료 이력 추적 불가

3. **병원 선택의 어려움**
   - 전문 분야 파악 어려움
   - 진료 가능 시간 불명확
   - 신뢰할 수 있는 리뷰 부족

4. **응급 상황 대응**
   - 24시 병원 찾기 어려움
   - 야간/주말 진료 정보 부족
   - 응급 상황 판단 기준 불명확

#### 1.3.2 솔루션 제공 가치

**Pet to You의 솔루션**:

1. **3초 예약**
   - 모바일에서 언제든 즉시 예약
   - 실시간 예약 가능 시간 확인
   - 자동 알림 및 리마인더

2. **올인원 건강 관리**
   - 모든 진료 기록 한곳에서 관리
   - 예방접종 스케줄 자동 추적
   - 건강 이력 기반 맞춤 추천

3. **스마트 병원 매칭**
   - AI 기반 병원 추천
   - 전문 분야, 거리, 리뷰 기반 필터
   - 실시간 진료 가능 여부 확인

4. **24/7 응급 지원**
   - 24시 응급실 실시간 검색
   - 응급 상황 판단 가이드
   - 긴급 예약 우선 처리

#### 1.3.3 경쟁 우위

**차별화 포인트**:

| 기능 | Pet to You | 경쟁사 A | 경쟁사 B |

| 전문 분야 필터 | ✅ | ❌ | ❌ |
| 24시 응급실 검색 | ✅ | ❌ | ❌ |
| 다중 반려동물 관리 | ✅ | ✅ | ❌ |
| 병원 리뷰 시스템 | ✅ | ✅ | ✅ |
| 예약 알림 | ✅ | ✅ | ✅ |

**핵심 경쟁 우위**:
1. **의료 전문성**: 수의사 자문단과 협력한 신뢰성 높은 정보

### 1.4 프로젝트 범위 및 제약사항

#### 1.4.1 MVP 범위 (Phase 1)

**포함 기능 (In Scope)**:

✅ **핵심 기능**:
- 사용자 인증 (카카오톡, Apple, 네이버 소셜 로그인)
- 병원 검색 (위치 기반, 필터링)
- 예약 시스템 (예약 등록, 확인, 취소)
- 반려동물 프로필 관리
- 건강수첩 기본 기능
- 예약 알림

✅ **필수 화면**:
- 로그인 화면
- 홈 화면
- 병원 검색 화면
- 예약 플로우 (반려동물 선택 → 예약 등록 → 확인 → 완료)

#### 2.2.2 기능 상세 명세

**F-001: 소셜 로그인**

**개요**: 3가지 소셜 로그인 옵션 제공

**로그인 옵션**:
1. **카카오톡 로그인**
   - Kakao SDK 사용
   - 필수 정보: 이름, 이메일, 프로필 사진

   - Sign in with Apple
   - 필수 정보: Apple ID
   - 이메일 숨기기 기능 지원

3. **네이버 로그인**
   - Naver SDK 사용
   - 필수 정보: 이름, 이메일

**게스트 모드**:
- 로그인 없이 앱 둘러보기 제공
- 제한 기능: 검색만 가능 (예약 불가)
- 게스트 → 회원 전환 유도

#### 2.2.3 수락 기준 (Acceptance Criteria)

**Given**: 사용자가 로그인 화면에 진입했을 때

**When**: 카카오톡 로그인 버튼을 클릭하면

**Then**:
- 카카오톡 앱/웹으로 리다이렉션
- 사용자 동의 후 프로필 정보 수신
- JWT 토큰 발급 및 저장
- 홈 화면으로 이동

**검증 항목**:
- ✅ 3가지 소셜 로그인 모두 정상 작동
- ✅ 로그인 실패 시 에러 메시지 표시
- ✅ 게스트 모드 접근 가능
- ✅ 로그인 상태 유지 (7일)

#### 2.2.4 기술적 고려사항

**인증 플로우**:
```
[앱] → [소셜 로그인 SDK] → [OAuth 2.0] → [백엔드 API] → [JWT 발급] → [로컬 저장] → [홈 화면]
```

**보안**:
- JWT 토큰 만료: 7일
- Refresh Token: 30일
- 토큰 저장: Secure Storage (Keychain/Keystore)

**에러 처리**:
- 네트워크 오류: 재시도 유도
- SDK 오류: 앱 로그인 페이지로 폴백
- 서버 오류: 에러 메시지 표시

#### 2.3.2 검색 알고리즘

1. **거리 기반** (기본)
   - 사용자 현재 위치에서 가까운 순
   - 반경 10km 이내 우선 표시

   - 리뷰 수 고려 (신뢰도)

**검색 로직**:
```
score = (distance_score * 0.4) + (rating_score * 0.3) + (open_status_score * 0.3)

distance_score = 1 - (distance / max_distance)
rating_score = rating / 5.0
open_status_score = is_open ? 1.0 : 0.3
```

   - UI: 다중 선택 체크박스

```
I want to 내 반려동물 정보를 기반으로 맞춤 보험을 추천받고
So that 복잡한 보험 상품을 쉽게 이해하고 선택할 수 있다
```

**US-021: 보험금 청구 지원**
```
As a 보험 가입자,
I want to 앱에 저장된 진료 기록으로 보험금을 간편하게 청구하고
So that 복잡한 서류 작업 없이 빠르게 보상받을 수 있다
```

#### 2.10.2 시장 현황 및 기회

**펫 보험 시장 규모**:
- 국내 펫 보험 시장: 약 1,000-1,500억원 (2024년 추정)
- 성장률: 연 25-30% (고성장)
- 가입률: 10-15% (매우 낮음, 선진국 대비 1/3 수준)

**시장 기회**:
```
반려동물 1,546만 마리
× 보험 미가입률 85%
× 평균 보험료 월 3만원
────────────────────────
잠재 시장: 연 4,700억원
```

**보호자 Pain Points**:
- 높은 의료비 부담 (연 평균 60-100만원)
- 보험 상품 복잡함 (비교 어려움)
- 가입 절차 번거로움
- 보험금 청구 복잡 (서류 많음)

#### 2.10.3 기능 상세 명세

**F-501: 보험 상품 비교**

**비교 항목**:
| 항목 | 상품 A | 상품 B | 상품 C |
|------|--------|--------|--------|

| 연간 보장 한도 | 500만원 | 700만원 | 1,000만원 |
| 자기 부담금 | 20% | 15% | 10% |
| 대기 기간 | 30일 | 60일 | 30일 |
| 보장 범위 | 질병+사고 | 질병+사고+예방 | 전체 |

**제휴 보험사**:

**비교 기능**:

**F-502: 맞춤 보험 추천**

**추천 알고리즘**:

**입력 정보**:
1. 반려동물 정보
   - 종류 (강아지/고양이)
   - 품종 (품종별 유전병 고려)
   - 나이 (노령견은 보험료 높음)
   - 기존 질병 유무

2. 보호자 정보
   - 월 예산
   - 보장 범위 선호 (질병만 vs 종합)
   - 자기 부담금 선호도

**추천 로직**:
```
score = (price_match * 0.3) + (coverage_match * 0.4) + (breed_risk * 0.2) + (popularity * 0.1)

price_match = 1 - |user_budget - premium| / user_budget
coverage_match = 보장 범위 일치도
breed_risk = 품종별 유전병 위험도 반영
popularity = 가입자 수 기반 인기도
```

**추천 결과**:
- TOP 3 추천 상품
- 추천 사유 명시
- 예상 연간 보장액 계산

**F-504: 보험사 연결** (간편 가입)

**가입 플로우**:

**Step 1: 상품 선택**
- 비교 또는 추천에서 상품 선택
- "가입하기" 버튼 클릭

**Step 2: 정보 자동 입력**
- 반려동물 정보 자동 연동
  - 이름, 품종, 나이, 성별
  - 건강수첩 데이터 (예방접종, 질병 이력)
- 보호자 정보 자동 입력
  - 이름, 연락처, 주소

**Step 3: 보험사 사이트 연결**
- 보험사 제휴 링크로 이동
- 사전 입력된 정보 전달 (API 연동)
- 최종 가입은 보험사 사이트에서 완료

**Step 4: 가입 완료 확인**
- 가입 완료 시 Pet to You로 정보 회신
- 보험증권 번호 저장
- 건강수첩에 보험 정보 표시

---

**F-505: 보험금 청구 지원** ⭐ 핵심 차별화

**청구 프로세스**:

**Step 1: 진료 기록 선택**
- 건강수첩에서 청구할 진료 기록 선택
- 다중 선택 가능 (한 번에 여러 건 청구)

**Step 2: 자동 서류 생성**
- 진료 기록 → 청구서 자동 변환
  - 진료 날짜, 병원, 진단명
  - 치료 내용, 처방 내용
  - 진료비 (영수증)
- PDF 자동 생성

**Step 3: 보험사 제출**
- 이메일 제출 (보험사 지정 주소)
- 앱 내 제출 (API 연동, 보험사 지원 시)
- 카카오톡 제출

**Step 4: 청구 내역 추적**
- 청구 상태: 접수 → 심사 중 → 승인 → 지급 완료
- 예상 지급일 표시
- 지급 금액 확인

**제공 가치**:
- 서류 작업 시간 **90% 단축** (30분 → 3분)
- 청구 성공률 향상 (누락 항목 자동 체크)
- 보험 활용도 증가

### 4.2 기술 스택 상세

#### 4.2.1 프론트엔드 기술 스택

**React Native vs Flutter 비교**:

| 항목 | React Native | Flutter |
|------|--------------|---------|
| 언어 | JavaScript/TypeScript | Dart |
| 성능 | 네이티브에 가까움 | 네이티브급 |
| 개발 속도 | 빠름 (웹 개발자 친화적) | 빠름 |
| UI 커스터마이징 | 높음 | 매우 높음 |
| 생태계 | 매우 풍부 | 성장 중 |
| 네이티브 모듈 | 풍부 | 제한적 |
| 학습 곡선 | 낮음 (JS 경험자) | 중간 (Dart 학습 필요) |

**추천**: **React Native**
- **이유**:
  - 개발팀의 JavaScript/TypeScript 경험 활용
  - 풍부한 라이브러리 생태계
  - 소셜 로그인 SDK 지원 우수
  - 빠른 MVP 개발 가능

**핵심 라이브러리**:
```json

    "react": "18.2.0",
    "react-native": "0.73.0",
```

#### 4.2.4 캐싱 전략 (Redis)

**캐싱 대상**:
1. **병원 검색 결과** (TTL: 5분)
   - 키: `hospitals:search:{lat}:{lng}:{filters}`
   - 목적: 반복 검색 성능 향상

2. **병원 상세 정보** (TTL: 1시간)
   - 키: `hospital:{id}`
   - 목적: 상세 페이지 로딩 속도 개선

3. **사용자 세션** (TTL: 7일)
   - 키: `session:{userId}`
   - 목적: 인증 상태 유지

4. **알림 카운트** (TTL: 실시간)
   - 키: `notifications:unread:{userId}`
   - 목적: 실시간 알림 카운트

**캐시 무효화 전략**:
- 병원 정보 수정 → 해당 병원 캐시 삭제
- 예약 생성 → 병원 가용 시간 캐시 삭제
- 리뷰 작성 → 병원 평점 캐시 삭제

### 4.4 API 설계 가이드라인

**API 명세 참조**: [api/api-spec.yaml](./api/api-spec.yaml)

**설계 원칙**:
1. **RESTful**: 리소스 기반 URL 설계
2. **Versioning**: URL에 버전 포함 (`/v1`)
3. **Pagination**: 리스트 API는 페이지네이션 필수
4. **Rate Limiting**: 사용자당 분당 100 요청
5. **HTTPS Only**: 모든 통신 암호화

**HTTP 메서드**:
- `GET`: 조회
- `POST`: 생성
- `PATCH`: 부분 수정
- `DELETE`: 삭제

**HTTP 상태 코드**:
- `200 OK`: 성공
- `201 Created`: 생성 성공
- `204 No Content`: 성공 (응답 본문 없음)
- `400 Bad Request`: 잘못된 요청
- `401 Unauthorized`: 인증 실패
- `403 Forbidden`: 권한 없음
- `404 Not Found`: 리소스 없음
- `409 Conflict`: 충돌 (예: 예약 시간 중복)
- `500 Internal Server Error`: 서버 오류

**에러 응답 형식**:
```json
{
  "code": "VALIDATION_ERROR",
  "message": "Invalid input data",
  "details": {
    "field": "email",
    "issue": "Invalid email format"
  }
}
```

### 4.5 보안

#### 4.5.1 인증 & 인가

**인증 전략**:
1. **소셜 로그인**: OAuth 2.0
   - Kakao, Apple, Naver SDK
   - Redirect URI 검증

2. **JWT 토큰**:
   - Access Token: 7일 만료
   - Refresh Token: 30일 만료
   - HS256 알고리즘

**인가 정책**:
- 사용자는 자신의 데이터만 접근 가능
- 병원 관리자는 자신의 병원 예약만 조회 가능
- 관리자는 모든 데이터 접근 가능

---

#### 4.5.2 데이터 보호

**암호화**:
- **전송 중 암호화**: TLS 1.3
- **저장 시 암호화**:
  - DB: AES-256 (RDS 암호화)
  - S3: AES-256 (서버측 암호화)
  - 비밀번호: bcrypt (cost factor 12)

**개인정보 보호**:
- 이메일, 전화번호: 암호화 저장
- 민감 정보 로깅 금지
- GDPR 준수 (삭제 요청 처리)

#### 4.5.3 보안 위협 대응

**OWASP Top 10 대응**:

1. **Injection**
   - Prisma ORM 사용 (parameterized queries)
   - Input Validation (class-validator)

2. **Broken Authentication**
   - JWT 토큰 보안 저장 (Secure Storage)
   - 토큰 갱신 메커니즘
   - 세션 타임아웃

3. **Sensitive Data Exposure**
   - HTTPS 강제
   - DB 암호화
   - 로그에 민감 정보 제외

4. **XML External Entities (XXE)**
   - JSON만 사용 (XML 미사용)

5. **Broken Access Control**
   - 인가 가드 (Authorization Guards)
   - 리소스 소유권 검증

6. **Security Misconfiguration**
   - 환경 변수 관리 (.env)
   - CORS 설정
   - Security Headers

7. **XSS**
   - Input Sanitization
   - Content Security Policy

8. **Insecure Deserialization**
   - JSON Schema Validation
   - 신뢰할 수 없는 데이터 검증

9. **Using Components with Known Vulnerabilities**
   - 정기 의존성 업데이트
   - npm audit
   - Dependabot

10. **Insufficient Logging & Monitoring**
    - CloudWatch 로깅
    - 보안 이벤트 추적
    - 알람 설정

## 6. 시장 분석 요약

> 상세 내용은 [market-analysis.md](./market-analysis.md) 및 [business-model.md](./business-model.md) 참조

### 6.1 시장 규모 및 기회

**전체 반려동물 시장**:
- 2024년: 8.5조원
- 2032년: 21조원 (2.5배 성장)
- CAGR: 7-8%

**타겟 시장 (TAM)**:
- 라이프케어: 0.8조원
- 애견 유치원: 3,000-5,000억원
- 입양 시장: 600억원
- **Total**: 약 9,276억원

**성장 동인**:
- 맞벌이 가구 증가
- 반려동물 보유 가구: 1,546만 (29.9%)
- 프리미엄 서비스 수요 증가
- 월평균 양육 비용: 19.4만원 (+26% YoY)

---

### 6.2 핵심 페인포인트 & 시장 기회

#### 페인포인트 #1: 정보 비대칭성
- **문제**: 가격 정보 90% 미공개, SNS 의존
- **영향**: 중도해지율 40%, 의사결정 3-5배 지연
- **기회**: 정보 투명성 개선 시 시장 15-25% 확대 (450-800억원)

#### 페인포인트 #2: 신뢰도 부족
- **문제**: 가짜 보호소 파양비 착취 (건당 600만원), 자격 미검증
- **영향**: 입양율 40% (낮음), 펫샵 구매 증가 (+29% in 4년)
- **기회**: 신뢰도 검증으로 입양율 40% → 65% (+25%p, 500억원 시장)

#### 페인포인트 #3: 예약 시스템 부재
- **문제**: 100% 수동 예약 (전화/카톡)
- **영향**: 운영 비효율, 예약 충돌
- **기회**: 자동화로 매출 +20-25% 증가

#### 페인포인트 #4: 실시간 모니터링 부재 (Phase 2)
- **문제**: CCTV 있어도 실시간 접근 불가
- **영향**: 보호자 불안감
- **기회**: 프리미엄 서비스로 매출 +20-30% (900억원)
